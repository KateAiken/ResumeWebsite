Morse Code Sender & Receiver


As my final project for MREN174, Data Structures, I built a morse coder translator that can send and receive morse code with other units. The hardware consists of an arduino with an LCD screen connected to a transmotter/receiver, LED and buzzer. The code used several different data structures including a binary search tree to assign combinations of dots and dashes to a letter, a queue to manage incoming and outcoming messages and a stack to manage the sorting of words or sentences into letters. The system can take an input of either letters or dots and dashes and outputs both to the other units once sent.

Louie - Mini Bot


I got the opportunity to work on the mini bot as part of my first year project course MREN103, Mechatronics Design I. We were tasked to design a line following algorithm so the bot could travel on a line around a board, pick up as many wire connectors as possible, return to the line, travel around the board and dump them in a bucket on the other side. This project involved a lot of tuning and the usage of several sensors including sharp sensors and reflective sensors. The goal of the project left room for creative solutions when faced with challenges. One of the biggest challenges we faced was turning a full circle once we were near the wall so we moved the sharp sensor to the bucket side of the robot and designed a backwards line following code. This allowed us to use the reflective sensor to aid in sensing when we had turned 90 degrees and start reversing, pushing the basket straight into the wire connectors.

Autimated Pre-Treatment Water System Prototype


The purpose of the prototype is to autonomously generate a minimum of 50 mL of clean water from approximately 250 mL of dissolved-bentonite water in under 10 minutes. All water is transported between tanks through a series of tubes. A submerged centrifugal pump pumps dirty water from a source tank into a coagulation tank. To assist in flocculation a small mixing bar is spun by magnetic interaction. A pair of small strong magnets are set into the mixing bar with the other set embedded in a gear of a gearbox. This gear box is located below the coagulation tank and reduces the RPM of a DC motor while increasing the torque. Simultaneously, approximately 0.8 g of alum is released into the tank by an alum dispenser, Figure 14. Once a turbidity sensor, located in the mixing tank, detects a sufficient voltage reading, the clear water above the sedimented layer is extracted by a peristaltic pump. It is then pumped up through a filter contained in a syringe. The filter consists of a filter sponge, silica sand, multi-purpose sand, and a small mesh strainer, which prevents the sand from floating up. Upon passing through the filter, the now-clean water exits the top of the syringe and descends into a final small tank where an additional turbidity sensor takes a reading to verify clarity. The clamshell container encloses the electronics to protect them from water damage. A breadboard placed in an accessible and visible location has push buttons to start and stop operation of the prototype. Status LEDs on the breadboard communicate the current state of the filtration process.

Fluidized Air Bed


As my final lab in Grade 12 Physics my partner and I were tasked with showing the “Magic of Physics” in a presentation to younger students in my school. We decided to build a fluidized airbed which is a box that pumps air into sand making the sand act like liquid. This was a difficult project as it was very precise to build and also needed very fine sand to work properly. It ended up working quite well as you can see in the video where when the weight is visible the air is turned off and the sand is acting normally. Once the air turns on the tennis ball floats up and the weight sinks showing the massive change in the sands density.

Waldo 16 - Underwater ROV


Waldo 16 was built by my school's Marine Advanced Technology Education (MATE) Remote Operated Vehicle (ROV) team for our second year attending the competition in Newfoundland. Waldo 16 was designed with four motors to control its movement on the XY-axis allowing it to move forwards/backwards, turn left/right and strafe left/right and two motors to move it up and down on the Z-axis. It was also equipped with two cameras and two claws, one vertical and one horizontal to give the driver a visual on the front and back of the robot making it more versatile. The year Waldo 16 was built I was acting as both the Head Engineer and Lead Software and Electrical Engineer. As the head engineer I worked on making sure the ROV worked and all parts functioned together for example making sure the motors were attached in the correct place so the strafing code worked correctly. As Lead Software and Electrical engineer I designed and assembled the electrical system and wrote the ROVs code while also mentoring a younger student to take on my role when I graduated.

Hangman Python Code


I wrote code for hangman as a part of my Grade 11 Computer Science class. This was a python based project taking advantage of ACSII style art to provide a visual of a hangman figure. The code uses lists to contain words with multiple different themes that the user can select. Then it sets up the game with empty spots with the number of letters and an empty hangman stand. As the user guesses letters the game uses for loops to check if they are correct and output the corresponding information to the user. The game also recorded the number of guesses and indicates a fail if the user does not guess after 6 tries. The user can input either a letter or a word as a guess making the game very versatile and the game can handle upper and lower case letters. One of the interesting features of this game is its unbreakable meaning you can't enter anything that will cause the code to fail including letters, symbols etc. This feature uses the characters decimal code to make sure it is in the range of lower or upper case characters.

2048 Java and Python Code


I wrote code for the game 2048 as my final project in my Grade 11 Computer Science class. The game 2048 is a classic number game where you have a four by four grid and a two block is generated into an empty spot in the grid after every move. The goal is to combine two of the same number together which adds the numbers and get as high a number as possible or eventually the number “2048”. This was a very difficult project to code as it was also my first time working on graphics with the PyGame library. In grade 11 I had a fully working graphic interface for the game but the game only progressed to combining to 16 because I had an inefficient structure for checking what blocks existed where. In grade 12 I attacked the project again but with more experience and in Java and was able to use 2D arrays to recreate the game. I started working on translating that code into python to work with the graphics and had the graphics partially working before the submission date.

Super Sink


In today's world, water usage is a huge concern. In remote or underdeveloped countries, access to safe/clean water can be almost impossible to find. Is it possible to create a compact and accessible system to reuse the water? In this project, we designed and built a sink to filter out particles and bacteria in the water. With research, we created a formula to simulate handwashing. Our testing criteria consisted of standard water tests to check the safety of the water. By running the handwashing solution through the filter, we learned the filter could only filter physical things out of the water, not chemicals.
